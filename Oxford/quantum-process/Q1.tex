\section{Problem 1}

The \textit{Crystal Code} with $2m + 2$ physical qubits has stabilizers generated by $\vec{X}= X_1 X_2 \cdots X_{2m + 2}$ and $\vec{Z} = Z_1 Z_2 \cdots Z_{2m + 2}$.
Its logical operators are $\X_j = X_j X_{2m+1}$ and $\Z_j = Z_j Z_{2m+2}$ for $j \in \{1, 2, 3, \cdots, 2m\}$.

\subsection{Commutativity Relationship}

We have the following lemma describing the commutativity relationship between all $X$ and all $Y$ Pauli.

\begin{lemma}\label{XZ-commute}
	For Pauli $\vec{X} = X^{v_1}\otimes X^{v_2} \otimes \cdots \otimes X^{v_n}$ and $\vec{Z} = Z^{w_1}\otimes Z^{w_2} \otimes \cdots \otimes Z^{w_n}$,
	where $\bm{v} = [v_1, v_2, \cdots, v_n]^T, \bm{w} = [w_1, w_2, \cdots, w_n]^T$ and $\bm{v}, \bm{w} \in F_2^n$, we have 
	$$
	\vec{X} \vec{Z} = (-1)^{\bm{v} \cdot \bm{w}} \vec{Z} \vec{X}
	$$

	In our notation, $X^0 = Z^0 = I$.
\end{lemma}

\begin{proof}
	We only need to realize that $XZ = -ZX$ and that $\X \Z = (X^{v_1}Z^{w_1}) \otimes (X^{v_2}Z^{w_2}) \otimes (X^{v_n}Z^{w_n})$.
\end{proof}

Another way to stated this lemma is that $\vec{X} \vec{Z} = \vec{Z} \vec{X}$ if and only if there are even number of $k$ such that $v_k = w_k = 1$.
We see $\vec{X}$ and $\vec{Z}$ has an overlap at index $k$ if $v_k = w_k = 1$ (That is, in the $k$th term they have repectively $X$ and $Z$).


Apply lemma \ref{XZ-commute} we see that all $\X_i$ commute with all $\vec{Z}$ and all $\Z_i$ commute with $\vec{X}$.
For $i \neq j$, there are no overlap between $\X_i$ and $\Z_i$, so they commute. If $i = j$, there is one overlap, so they anti-commutes.
This is the desired commutivity relationship for CSS code.

The error $P$ which is only none-trivial in the $i$-th qubit does not commute with $\vec{X}$ and $\vec{Z}$ simultaneously, as there is no one-qubit Pauli that commutes with $X$ and $Z$ at the same time.
The error $X_1X_2$, however, commutes with both of $\vec{X}$ and $\vec{Z}$.
So the depth of this code is 2.

Since we have 2 stabilizers, the logical qubits are $2m+2 -2 = 2m$.

So crystal code has signature $[[2m+2, 2m, 2]]$.

\subsection{Fault Tolerance}

When it comes to single qubit errors, we only need to consider $X$ and $Z$ errors, as $X, Z, -iZX = Y,$ and $I$ forms a basis of all one-qubit unitary matrices, and any measurement will collapse the error to $X$ or $Z$.
Moreover, since we have isomorphic $X$ forms and $Z$ forms if disregarding the colour, all analysis on $X$ error on $X$ forms can be applied to $Z$ error on $Z$ forms, and vice versa.
So we only need to consider $X$ error.

The single qubit logical operations are transverse thus fault tolerant. 
We can show it explicitly in the following diagram, where one $X$ error propagates to at most one $X$ error after the operation.
\ctikzfig{one-qubit-error}

One $X$ error before the CNOT operation, however, can propagate to two. Thus the CNOT operation is not fault tolerant.

The simultaneous H gates is fault tolerant as, as it is transverse (as shown in section 12.3 of \cite{kissinger}).
To justify it in a different way, consider simultaneous $H$ gates on the logical qubits, which is equivalent to simultaneous $H$ gates on the physical qubits and swap of the last two physical qubits.
A single $H$ gate on the physical qubit is definitely fault tolerant. 
The swap, considered as three consecutive CNOT gates, is also fault tolerant as long as there is no errors between the three CNOT gates.
(In addition, since swap is literally swapping the two qubits, it can be down physically if ignoring any quantum entanglements.)
